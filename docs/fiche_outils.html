Questions fréquentes du mentor
Réponses préparées — à connaître
Pourquoi PlantUML plutôt que Draw.io ou Lucidchart ?
PlantUML génère des diagrammes depuis du texte pur (.puml) — versionnables dans Git comme du code. Draw.io produit des binaires XML non versionnables. C'est le standard industrie pour les projets qui évoluent dans le temps.
Comment avez-vous généré les PDF ?
Documents écrits en HTML/CSS pour contrôle total du rendu. WeasyPrint (Python) les convertit en PDF. Images UML encodées en base64 dans le HTML → PDF auto-portant, aucun fichier externe manquant.
Pourquoi Django Channels et pas Socket.io ?
Channels est l'extension officielle de Django pour WebSocket — pas de second service Node.js. Transforme Django de WSGI (synchrone) en ASGI (asynchrone, WebSocket). Redis = standard téléphonique entre workers.
ASGI vs WSGI — quelle différence ?
WSGI : 1 requête → 1 réponse → connexion fermée. ASGI : connexion persistante, le serveur envoie des données sans demande du client. C'est ce qui rend le WebSocket possible. Daphne = serveur ASGI officiel de l'équipe Channels.
Pourquoi PostgreSQL et pas MongoDB ?
L'audit a montré que MongoDB sans schéma strict a créé des données incohérentes dans l'existant. PostgreSQL garantit les transactions ACID (si une réservation échoue, la base revient à son état antérieur). ORM Django natif premium.
Pourquoi author est nullable dans Message ?
Conformité RGPD. On ne supprime pas les messages (historique utile pour les litiges). SET_NULL Django : à la suppression du compte, author = NULL → «Utilisateur supprimé». C'est de l'anonymisation, pas de la suppression.
Qu'est-ce que la PoC prouve exactement ?
1) Django + Channels + Redis permet le temps réel WebSocket. 2) Messages persistés en PostgreSQL via le modèle Message. 3) Structure du code prête à être étendue. La PoC ne prouve pas l'UI — volontairement limitée au tchat.
